  DEPLOYMENT WORKFLOW

STEP 1: Push to GitHub
  $ git add .
  $ git commit -m "Production deployment ready"
  $ git push origin main

STEP 2: Deploy Backend (Railway)
  $ railway login
  $ railway init
  $ railway up
  → Add environment variables in dashboard
  → Add volume at /app in dashboard

STEP 2 (Alternative): Deploy Backend (Render)
  → Go to render.com
  → New Web Service
  → Connect GitHub repo
  → Fill environment variables
  → Deploy (disk auto-configured)

STEP 3: Get Backend URL
  → Copy from Railway/Render dashboard
  → Example: https://kisaan-ai-production.up.railway.app

STEP 4: Update Backend Environment
  → Set VITE_API_URL=<backend-url>
  → Redeploy backend

STEP 5: Deploy Frontend (Vercel)
  $ vercel --prod
  → Set VITE_API_URL=<backend-url>

STEP 6: Test Deployment
  $ bash verify-deployment.sh <backend-url>
  → Or manually test endpoints

───────────────────────────────────────────────────────────────────────────────
  TROUBLESHOOTING GUIDE
───────────────────────────────────────────────────────────────────────────────

ISSUE: Weather shows 0°C
  → Check OPENWEATHERMAP_API_KEY is set correctly
  → Verify API key is valid at openweathermap.org

ISSUE: Image upload fails with 422 error
  → Verify python-multipart==0.0.12 in requirements.txt
  → Do NOT upgrade starlette or fastapi

ISSUE: Database resets on every deploy
  → Railway: Add persistent volume at /app
  → Render: Verify render.yaml has disk configuration

ISSUE: CORS errors in browser console
  → Verify VITE_API_URL matches backend URL exactly
  → Check backend logs for CORS middleware errors

ISSUE: "ChromaDB EMPTY" warning in logs
  → Run: python scripts/ingest_data.py
  → This indexes PDFs for RAG features (optional)
